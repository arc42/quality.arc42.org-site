Scenario: The BuDIC tool needs to be compatible with the latest versions of MySQL, PostgreSQL, Oracle and DB2.
If an update or new version of one of these databases is released by the respective vendor, BuDIC can be updated to be compliant with the new version within 2 person-days of development and test effort.

Background: Business data is stored in different databases (MySQL, PostgreSQL, Oracle, DB2). Integrity and consistency of certain data types need to be checked across these databases. An integrity checking tool (BuDIC) has been developed, that is able to access several datas ources in parallel.
