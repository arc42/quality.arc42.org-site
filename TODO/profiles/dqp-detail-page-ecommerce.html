01 E-commerce / Online Shop

Transactional systems for selling goods or services online. Quality priorities centre on fast response under peak load, reliable order processing, payment security, and conversion-optimised user experience.

mass-market light regulation web transactional business-critical
Change pace continuous

Variants

This base profile covers what all online shops share. Your product domain adds specific quality shifts:

Ticketing / Events

Konzerttickets, Flüge, Hotel

Extreme peak concurrency — thousands competing for the same item simultaneously. Fairness, queue management, and elastic scalability dominate.

elasticity fairness response time atomicity

Regulated goods

Medikamente, Alkohol, Waffen

Legal compliance shifts from "light" to "compliance-heavy." Age verification, prescription workflows, geo-restrictions, audit trails.

compliance auditability data protection access control

Custom / print-on-demand

Fotobücher, T-Shirt-Druck, Möbel

Long asynchronous fulfilment chains. Order status tracking, configurator usability, and data integrity across production steps.

traceability consistency operability configurability

Fashion / high-return

Kleidung, Schuhe, Accessoires

Return rates of 40–60% shape the system: reverse logistics integration, visual product presentation, size/fit recommendation quality.

accuracy interoperability usability appearance

Marketplace / multi-vendor

Amazon-Modell, Etsy, B2B-Plattform

Multi-tenancy, vendor isolation, trust/rating systems, split payments. Quality concerns overlap with the SaaS/B2B profile.

multi-tenancy isolation accountability scalability

Subscription / recurring

Abo-Boxen, SaaS-Lizenzen, Streaming

Billing cycle reliability, churn-prevention UX, usage metering, prorating and plan changes without data loss.

reliability data integrity flexibility transparency

Curated characteristics

Starting set for e-commerce — adapt to your context. Links go to full Q42 definitions.

Critical Typically non-negotiable for any online shop
  • Availability #reliable Downtime = lost revenue. 99.9%+ during peak hours.
  • Response time #efficient Page load <2s. Checkout <1s. Every 100ms costs conversion.
  • Data integrity #reliable Orders, inventory, payments must be consistent at all times.
  • Confidentiality #secure Payment data, personal data. PCI-DSS, GDPR compliance.
  • Authenticity #secure User identity, payment authorization, fraud prevention.
  • Accessibility #usable EAA / WCAG 2.1 AA — now legally required in EU for e-commerce.
Important Highly relevant for most online shops
  • Scalability #efficient Black Friday, flash sales, campaign peaks.
  • Recoverability #reliable Order state recovery after failures. No lost carts, no duplicate charges.
  • Usability #usable Checkout friction, search quality, mobile experience.
  • Interoperability #flexible Payment gateways, shipping providers, ERP, PIM, CRM integration.
  • Deployability #operable Zero-downtime deployments. Feature flags for A/B testing.
  • Monitorability #operable Real-time conversion funnels, error rate alerts, performance budgets.
Consider Relevant depending on variant and scale
  • Elasticity #efficient Auto-scale under unpredictable load spikes. See variant: Ticketing.
  • Configurability #flexible Product configurators, personalization, A/B test infrastructure.
  • Compliance #suitable Pharma, alcohol, weapons — see variant: Regulated goods.
  • Testability #maintainable Checkout flows, payment integration, multi-browser testing.
  • Internationalization #suitable Multi-language, multi-currency, locale-specific tax rules.

Example requirements

Concrete, measurable starting points — drawn from Q42's requirements collection.

Product pages load within 1.5 seconds (P95) under normal load and within 3 seconds during peak events (10× baseline traffic), measured at the CDN edge.
#efficient → Response time (web)
The ordering system processes each checkout as an atomic transaction: either the complete order (including payment capture and inventory reservation) succeeds, or no state change persists. Zero partial orders.
#reliable → Atomicity
No unencrypted payment card data on persistent storage. Zero PAN values stored outside the PCI-DSS compliant payment processor's scope.
#secure → Confidentiality (payment)
A first-time user completes a purchase (from product page to order confirmation) in under 4 minutes, without requiring account registration.
#usable → Task efficiency
New code reaches production within 2 hours of merge to main. Rollback to previous version completes within 5 minutes.
#operable → Deployment frequency

Relevant standards

Standards most commonly applicable to e-commerce systems.

PCI-DSS
Payment card data security. Required for all systems handling card payments.
GDPR
EU data protection. Customer data, consent management, right to deletion.
WCAG 2.1 / EAA
Web accessibility. Legally required for EU e-commerce from 2025.
ISO/IEC 27001
Information security management. Often required by enterprise clients / B2B.
ISO/IEC 25010
Product quality model. The foundational standard behind Q42's structure.
