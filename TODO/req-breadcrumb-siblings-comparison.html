How the ranking would work in Liquid

{% raw %}{%- comment -%} For each candidate sibling, score against the current page. {%- endcomment -%}
{%- assign current_related = page.related -%}
{%- assign current_tags = page.tags -%}
{%- assign scored = "" | split: "" -%}

{%- for req in site.requirements -%}
  {%- if req.permalink == page.permalink -%}{%- continue -%}{%- endif -%}
  {%- assign rel_overlap = req.related | intersection: current_related | size -%}
  {%- assign tag_overlap = req.tags | intersection: current_tags | size -%}
  {%- assign score = rel_overlap | times: 5 | plus: tag_overlap -%}
  {%- if score > 0 -%}
    {%- assign entry = req.permalink | append: "|" | append: score | append: "|" | append: rel_overlap -%}
    {%- assign scored = scored | push: entry -%}
  {%- endif -%}
{%- endfor -%}

{%- comment -%} Sort lex descending by the "score" segment, then slice top 8. {%- endcomment -%}
{%- assign top = scored | sort | reverse | slice: 0, 8 -%}{% endraw %}

~30 lines of Liquid total in _layouts/requirements.html. The breadcrumb resolution adds another 8 lines (look up page.related[0] in site.qualities by permalink slug to get the title). No JavaScript, no new dependencies.
